# Comandos básicos GIT >Para hacer uso de los siguientes comandos, es necesario contar con la herramienta(git) instalada. ## 1. git config Finalidad: configurar los ajustes de git, como el nombre de usuario y el correo electrónico. A continuación se muestra un ejemplo para configurar la **identidad del usuario**: ``` git config --global user.name "Tu nombre" ``` ``` git config --global user.email "Tu correo electrónico" ``` La siguiente instrucción ayuda a verificar la configuración actual ``` git config --list ``` ## 2. git init Finalidad: inicializar un nuevo repositorio GIT. A continuación se muestra un ejemplo: ``` git init``` ## 3. git clone Finalidad: clonar un repositorio existente. A continuación se muestra un ejemplo: ``` git clone https://github.com/usuario/repositorio.git``` ## 4. git status Finalidad: mostrar el estado del directorio de trabajo y del área de preparación. A continuación se muestra un ejemplo: ``` git status``` ## 5. git add Finalidad: añadir el contenido de los archivos al índice (área de preparación). A continuación se muestra un ejemplo: ``` git add . (add all files)``` ## 6. git commit Finalidad: registrar los cambios en el repositorio local. A continuación se muestra un ejemplo: ``` git commit -m "Mensaje de confirmación"``` ## 8. git pull Finalidad: Obtener e integrar con otro repositorio o rama local. A continuación se muestra un ejemplo: ``` git push origin main``` ## 9. git push Finalidad: actualizar las referencias remotas junto con los objetos asociados. A continuación se muestra un ejemplo: ``` git push origin main``` ## 10. git log Finalidad: mostrar registros de las confirmaciones. A continuación se muestra un ejemplo: ``` git log```
